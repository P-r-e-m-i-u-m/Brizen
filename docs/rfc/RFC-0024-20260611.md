## Summary

This RFC proposes replacing our custom JWT authentication system with a standards-compliant OAuth 2.0 and OpenID Connect implementation to support third-party integrations and SSO.

## Motivation

Cannot support third-party integrations without sharing credentials. No SSO capability. Security audit recommended standards-based auth.

## Detailed Design

Grant types: Authorization Code + PKCE, Client Credentials, Refresh Token. Scopes: read/write profile and orders, admin:users restricted. Existing JWT sessions valid for 30 days during migration.

## Alternatives Considered

Auth0/Okta: fully managed but vendor lock-in and high cost. Keycloak: open source but significant operational overhead. Passport.js: library not a full standards implementation.
