## Executive Summary

JWT token validation intermittently failed under high concurrency due to a Redis SET race condition. Approximately 2,400 authenticated requests returned 401 during the incident window.

| Time | Event |

## Root Cause Analysis

The auth middleware used Redis SET without the NX flag, creating a race condition where concurrent requests could overwrite cached tokens mid-verification. The race condition required 500+ concurrent requests to manifest.

## Resolution

Rolled back to previous auth middleware version immediately. Developed fix using Redis SET with NX flag. Fix deployed after load testing at 2000 concurrent users with 0 failures.
